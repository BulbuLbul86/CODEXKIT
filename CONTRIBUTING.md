# Contributing

Спасибо за интерес к CODEXKIT. Проект работает с локальной рабочей средой пользователя, поэтому главный принцип вкладов — не добавлять в репозиторий личные данные и не расширять сбор секретов без явного предупреждения.

## Как делать PR

1. Создай ветку от актуального `main`.
2. Делай небольшие изменения с понятным описанием.
3. Не коммить результаты запуска CODEXKIT.
4. Перед PR запусти:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\prepublish-check.ps1
```

5. В описании PR укажи, что изменилось, как проверялось и есть ли влияние на приватность.

## Что нельзя добавлять

Не добавляй:

- `state/`;
- `repo-snapshots/`;
- `docs/`;
- `installers/`;
- transfer-архивы и `.sha256` от личных комплектов;
- `machine-info.json`;
- `environment-inventory.json`;
- токены, ключи, сертификаты, приватные конфиги;
- реальные пользовательские пути в `custom-paths.json`.

`custom-paths.json` в репозитории должен оставаться пустым шаблоном.

## Стиль PowerShell-кода

- Поддерживай Windows PowerShell 5.1+.
- Не используй возможности, доступные только в PowerShell 7.
- Предпочитай явные параметры и понятные сообщения на русском.
- Для операций с файлами используй `-LiteralPath`, если путь пришёл из данных пользователя.
- Не запускай установку программ и реальное восстановление в тестах.
- Для опасных действий добавляй dry-run или plan-поведение, если это возможно.

## Кодировка файлов

PowerShell-скрипты с русским текстом должны сохраняться в кодировке, совместимой с Windows PowerShell 5.1.

Для `.ps1` с кириллицей допускается UTF-8 with BOM. Это помогает Windows PowerShell 5.1 корректно читать русские строки и не ломать парсер после массового редактирования файлов.

После массового редактирования обязательно запускай:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\prepublish-check.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File .\restore-codexkit.ps1 -PlanOnly
Invoke-Pester .\tests
```
